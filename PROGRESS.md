## What the game does (v0.1)

- **Life stages / evolution:** Botamon → Koromon → Agumon → Greymon → **MetalGreymon** (well-cared) or **SkullGreymon** (neglected). Accelerated timing, all tunable via a single `GameConfig.gameSpeed`.
- **Care loop:** feed 🍖, clean 🧹, medicine 💊 (only when sick), play ❤️. Irrelevant buttons dim.
- **Needs & neglect:** hunger, mess (poop), sickness, happiness, and a hidden `careScore`. Neglect → sickness → death.
- **Death & restart:** death screen ("returned to the Digital World") → hatch a new egg from Botamon.
- **Reminders:** a background (workmanager) task that reads the saved pet read-only and only notifies when it genuinely needs attention.
- **Look:** LCD-green screen, original pixel-art UI icons, community Digimon character sprites, idle animation + scale-bounce feedback.

## Tech stack & why

| Choice | Reason |
|---|---|
| **Flutter + Flame** | Whole dev pipeline is CLI-drivable (create/test/build) — no GUI editor needed. Flame has native spritesheet/animation support. |
| `shared_preferences` (JSON) | Simple, codegen-free persistence for a single pet-state blob. |
| `flutter_local_notifications` + `workmanager` | Mature, Dart-only plugins for reminders + background scheduling. |

## Architecture

```
lib/
  main.dart            # app entry; inits workmanager + runs the game
  state/               # PURE, TESTED game logic (no Flutter/Flame deps)
    game_config.dart   #   all tuning constants + gameSpeed
    pet.dart           #   Pet model, enums, JSON, per-stat time anchors
    pet_logic.dart     #   applyElapsed, care actions, evolution, needsAttention
    pet_repository.dart#   persistence interface + shared_preferences impl
    notifications.dart #   flutter_local_notifications wrapper
    background.dart     #   workmanager entry point (read-only reminder)
  game/                # Flame rendering
    sprite_map.dart    #   stage→sheet + verified idle frame mapping
    pet_component.dart #   idle animation + reactBounce
    vpet_game.dart     #   ties logic+persistence+1s tick together
  ui/                  # Flutter screens
    home_screen.dart   #   canvas + care buttons + status icons + lifecycle
    death_screen.dart  #   death + restart
test/                  # 34 unit/widget tests (logic, repo, game, death screen)
assets/sprites/        # character sprite sheets (community art)
assets/ui/             # original pixel-art icons (generated by scripts/make_icons.py)
```

**Time model:** each time-driven stat carries its own "since" anchor timestamp, so the logic is
correct both for a single "reopen after absence" jump and the in-app 1-second tick (no lost
remainders). All logic functions take an explicit `nowMs` for deterministic tests.

## Key design decisions (from brainstorming)

- **MVP scope:** essential care loop only — no battles, no home-screen widget, no discipline system beyond `careScore`.
- **Dual Perfect branch** (MetalGreymon vs SkullGreymon) decided by accumulated `careScore`.
- **Accelerated time**, tunable from one constant.
- **Original UI art** (no Bandai device art/logo) — only the character sprites are from a community collection. Repo kept **private** given the third-party (Bandai) IP in the sprites.
- **Death is permanent** and restarts from the egg, giving weight to neglect.

## Notable gotchas fixed along the way

- **Corrupt pre-existing NDK** blocked builds → NDK requirement removed (app has no native code).
- **`flutter_local_notifications`** needs **core library desugaring** → enabled + `desugar_jdk_libs`.
- **Release APK crashed on launch** (`NoSuchMethodException: WorkDatabase_Impl.<init>`) because **R8** stripped a reflectively-loaded WorkManager class → **disabled R8 shrinking for release** (`isMinifyEnabled=false`). Debug builds don't run R8, so they were never affected.
